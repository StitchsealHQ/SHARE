0개요

서비스 정의와 핵심 원칙
서비스명StitchSeal (스티치실) — stitch(바느질) + seal(인장·봉인)
한 줄 정의패션 디자이너의 창작물을 공개 전에 블록체인으로 시점 증명하고, 디자인 출원과 B2B 거래까지 연결하는 원스톱 플랫폼
타깃법적 대응 여력이 없는 신진·독립 패션 디자이너 (1차), 브랜드·바이어·제조사 (2차)
현재 상태동작하는 프로토타입 라이브 — stitchsealhq.github.io (등록→해시 봉인→증명서→검증 전 과정 체험 가능)

1문제 인식 (Problem)

패션 디자인은 IP 보호의 사각지대다

1-1. 카피가 가장 빠른 산업

  • 카피 유통 속도: 런웨이·SNS 공개 직후 수 주 내 카피 제품이 시장에 유통된다. 신진 디자이너일수록 법적 대응 여력이 없어 피해가 집중된다.
  • 짧은 수명 주기: 시즌 단위로 소비되는 디자인 특성상, 권리 확보가 조금만 늦어도 보호의 실익이 사라진다.
  • 증거의 부재: 도용 분쟁의 승패는 "누가 먼저 창작·공개했는가"의 증거 싸움인데, 개인 디자이너는 체계적 증거를 남기지 않는다.

1-2. 법제는 있지만, 디자이너가 쓰기 어렵다

보호 수단내용한계
디자인보호법 등록의류는 일부심사 + 신속심사 6일. 유일한 독점권시즌마다 수십 룩을 전부 출원하기엔 비용·절차 부담
부정경쟁방지법 (자)목등록 없이도 출시 후 3년간 형태모방 금지 — 실전 무기창작·공개 시점의 증거가 없으면 무력
저작권응용미술저작물 인정 요건이 까다로움의류 디자인엔 보조적
빈칸 "공개 전 즉시, 저비용으로, 조작 불가능한 시점 증거를 만드는 인프라"

1-3. 왜 지금인가

  • 블록체인 타임스탬프 기술은 성숙 단계 — 해시 앵커링은 건당 수십 원 이하로 가능
  • AI 생성 콘텐츠 급증으로 "출처·창작자 증명" 수요가 구조적으로 확대
  • 신진 디자이너 육성·K-패션 진흥은 서울시·중앙정부의 지속 정책 과제 — 공공 재원과 정합성이 높음

2실현 가능성 (Solution)

시장의 실패에서 배운 설계 — 그리고 이미 동작하는 프로토타입

2-1. 시장의 실패에서 배운 설계 원칙

사례결과교훈
WIPO PROOF (세계지식재산기구 타임스탬프)2021년 중단세계 최고 공신력으로도 증명 단품은 팔리지 않는다
루이비통 AURA (명품 블록체인 정품인증)확산 중증명이 상거래 흐름 속에 심어질 때 작동한다
설계 원칙: 등록이 곧 영업이 되게 한다 등록된 디자인이 바이어에게 노출되고 거래로 이어지며, 그 과정에서 증거가 자동으로 쌓인다 — 디자이너가 돈을 내는 이유를 '보호'가 아닌 '매출'로 만든다.

2-2. 3단 가치제안 (정직한 포지셔닝)

"등록하면 보호됩니다"는 과장이다. StitchSeal은 기존 법제와 상호보완하는 3단 구조를 제공한다.

①공개 전 즉시 증거
창작 시점을 퍼블릭 체인에 봉인 (3초, 조작 불가능)
②정식 권리 연계
변리사 제휴로 디자인보호법 출원 원스톱 (신속심사 6일)
③도용 대응 지원
부경법 (자)목 + 체인 증거 리포트 + 내용증명 템플릿

2-3. 서비스 흐름 (사업 모델 4단계)

1
등록
시즌 컬렉션 업로드
2
증명
해시 체인 기록 · 창작 증명서 발급
3
권리화
변리사 제휴 · 디자인 출원 원스톱
4
거래
B2B 카탈로그 · 라이선싱·수주 연결

2-3b. 양면 플랫폼 설계 — 디자이너·바이어 계정 분리

④거래 단계는 두 번째 사용자군(바이어)을 전제한다. 두 역할은 권한이 다르므로 계정을 분리하되, 분리 자체보다 신뢰 설계가 핵심이다.

설계내용이유
바이어 승인제디자이너는 즉시 가입, 바이어는 사업자·브랜드 확인 후 승인 입장(1~2일 심사)최악 시나리오 차단 — 카피 업체가 바이어로 가입해 미공개 디자인을 열람하는 것. "검증된 바이어만 봅니다"는 디자이너 설득 포인트이자 바이어에겐 폐쇄형 소싱 채널의 가치
열람 로그 = 증거바이어가 어떤 디자인을 언제 열람했는지 기록봉인(창작 시점 증거) + 열람 로그(접촉 증거)가 결합된 증거 체인 — 열람한 바이어 측에서 유사 제품이 나오면 결정적 정황. 기록된다는 사실 자체가 도용을 사전 억제. 서비스 핵심 컨셉("모든 것이 증거가 된다")의 확장
공개 수준 통제룩별로 디자이너가 선택: 🔒비공개(기본) / 👔바이어 공개(워터마크·저해상도 프리뷰, 원본은 협의 후) / 🌐전체 공개미공개 시즌 = 영업비밀. "봉인 먼저, 노출은 선택"이 신뢰의 기반
계정 구조단일 인증 시스템 + 역할(designer/buyer) 분기, 가입 진입점만 분리별도 시스템은 관리 비용만 증가. 바이어 열람권 구독이 성장기 수익 모델과 직결

구현 시기: 실계정·승인 시스템은 Phase 3. 단, 바이어 카탈로그 데모는 프로토타입에 구현 완료 — ④거래까지 시연 가능.

2-4. 기술 구현 — 이미 동작한다

  • 프로토타입 완성·공개 (2026-07): stitchsealhq.github.io
    • 파일 업로드 → 실제 SHA-256 해시 생성(브라우저 내 계산, 원본 미전송) → 봉인 → 창작 증명서 발급 → 공개 검증(1바이트 변조도 검출) 전 과정 동작
    • 바이어 카탈로그 데모 (/buyers.html) — 승인제 안내, 워터마크 프리뷰, 열람 기록 표시까지 ④거래 단계 시연 (2-3b 설계 반영)
  • Next.js 앱 개발 중: 체인 앵커링 API 구현 완료 — 환경 설정만으로 실제 EVM 체인 기록 전환 가능 (ethers.js, EVM 호환 체인)
  • 낮은 기술 난이도가 곧 전략: 자체 메인넷 불필요, 해시만 저비용 퍼블릭 체인에 앵커링. "웹 포트폴리오 + 해시 기록 모듈" 수준 — 2~3인 팀, 수개월 내 정식 서비스 가능
  • 규제 리스크 최소화 설계: 코인 발행·토큰 판매·수익 배분 없음 → 가상자산사업자(VASP) 신고 대상 아님, ICO 금지 저촉 없음. 확장 대비 EVM 호환 기준 설계

2-5. 경쟁·대체재 대비 차별성

대안한계StitchSeal의 위치
특허청 신속심사 (6일)이미 싸고 빠르지만, 공개 '전' 즉시 증거는 아님대체가 아닌 보완 — 출원 연계로 오히려 특허청 접수를 늘림
단순 타임스탬프 서비스WIPO 선례 — 수요 없음증명을 거래(영업 자산)와 묶는 구조
포트폴리오 플랫폼 (Behance 등)공개 = 카피 노출 시작, 증거력 없음"공개 전 봉인"이라는 반대 지점
도매 플랫폼 등록 이력 (신상마켓)등록 = 즉시 13만 소매·경쟁 도매에 노출되는 공개 '후' 증거. 플랫폼 내부 데이터라 중립성·영속성 약함(삭제·폐업 시 소실). 브랜드 가품 차단은 있으나 형태모방(디자인 카피) 대응은 공백 — 카피 생산 도매도 플랫폼 고객이라 구조적으로 심판 불가"등록 전 봉인"으로 보완 — 플랫폼 중립 제3자 증거 레이어. "신상마켓에 올리기 전에 봉인하세요" = '쇼 전에 등록'의 동대문 버전

3성장 전략 (Scale-up)

3단계 로드맵 · 수익 모델 · 시장 규모 · 파트너십

3-1. 3단계 로드맵

Phase목표핵심 실행
1. 증거 금고 · 파일럿사용 습관 검증포트폴리오 등록 + 타임스탬프 + 증명서(PDF). 패션쇼 공간·창작 스튜디오 제휴로 파일럿 그룹 확보 — "쇼 전에 등록하고 무대에 올려라"
2. 법적 실효성 연결보호 가치 실증변리사 제휴 원스톱 출원, 도용 대응 패키지, 이미지 검색 모니터링. 분쟁 승소 사례 1건 확보 → 결정적 마케팅 자산
3. 거래로 확장수익 전환B2B 카탈로그(바이어·제조사 개방), 유통 파트너 제휴로 "등록→노출→수주" 완성, 라이선싱·로열티 정산 자동화

3-2. 수익 모델 (단계별)

  • 초기 (1~2년차) — 공공 재원 기반 무료 제공: 서울시 패션 정책사업·특허청 IP 지원·콘진원 과제 수주로 운영 (디자이너 지불 의사가 낮은 초기의 현실적 재원)
  • 성장기 — 출원 연계 수수료(변리사 제휴), 도용 모니터링 구독, 브랜드·기업 회원(B2B 카탈로그 열람권)
  • 성숙기 — 라이선싱 거래 중개 수수료. IP 토큰화(조각투자)는 디지털자산기본법 2단계 입법 확정 후 옵션 검토

3-3. 시장 규모

  • TAM: 국내 패션시장 49.5조 원 (2024, 한국섬유산업연합회)
  • SAM: 정부 창업·IP 지원 재원 — 중기부 창업지원 3.46조 원/508개 사업 (2026) 중 패션·IP 접점 사업 + 서울시 패션 정책 예산
  • SOM(초기): 서울 소재 신진 디자이너 파일럿 — 패션쇼·창작 스튜디오 제휴 채널 기준 [수요 인터뷰·조사로 구체화 예정]

3-4. 핵심 파트너십 — 성공의 전제 조건

진입장벽은 블록체인 기술이 아니라 "디자이너 풀 × 증명 × 유통"의 조합이다.

  1. 디자이너 커뮤니티 — 창작 스튜디오·패션쇼 공간·협회 제휴로 콜드스타트 해결
  2. 공공기관 — 서울시 패션 정책·특허청 IP 지원·콘진원 과제 연계 (정책 명분 뚜렷)
  3. 유통 파트너 — 도매 플랫폼·쇼룸·해외 리테일: 등록→수주의 출구
    • 구체안(신상마켓): 동대문 도매 80%·소매 13만이 쓰는 1위 B2B 플랫폼 — 매일 수만 개 신상 등록이 곧 공개 시점. 신상스튜디오(픽업→촬영→반납)처럼 '등록 전' 워크플로에 서비스가 끼어드는 선례가 이미 있어 "촬영 원본 해시 봉인 → 등록" B2B2C 제휴 시나리오 성립. 플랫폼 내 자체제작 도매(디자인 매장) = 페인 검증된 유력 고객군(수요 인터뷰 세그먼트 후보)
  4. 법률 파트너 — 변리사·법무 제휴: 증거가 권리·분쟁 대응으로 이어지는 신뢰 고리

  • 패션 창작 생태계 실무 (현직): 서울시 산하 서울패션창작스튜디오 운영 매니저 — 입주 디자이너 지원·홍보·정부사업 기획 참여. Phase 1 파일럿의 제휴 채널(창작 스튜디오·디자이너 풀)을 실무로 아는 위치
  • 패션 IP·브랜드 프로젝트 경험: 한국콘텐츠진흥원 지속가능패션 제작지원(2024, 총 5.9억), 종로구 공동브랜드 '일루셀'(창신동 봉제 고부가가치화) 참여 — 공공 재원 기반 패션 사업의 실행 문법 이해
  • 수출·기술 실행력: 호주 브랜드 KANZI 수출 실무(소싱·수출서류·매출분석) + ERP·AI 업무자동화 시스템(MCP) 직접 구축·운영 — 프로토타입을 직접 개발한 기술 실행력의 근거 (본 기획서의 라이브 프로토타입도 직접 제작)

※ 모든 데이터·사례는 공개 자료 기반. 소속 기관의 비공개 정보는 사용하지 않음.

5추진 일정

공모전 확정 시 예산과 함께 구체화
시기마일스톤
~1개월수요 검증: 디자이너 인터뷰 3~5건(도용 경험·등록 의향), 변리사 자문 1건(부경법 (자)목 증거 실무)
1~3개월정식 MVP: 실제 체인 앵커링 전환, 계정·컬렉션 관리, 배포(Vercel)
3~6개월파일럿: 창작 스튜디오·패션쇼 1곳 제휴, 파일럿 디자이너 10~20팀, 등록 100건
6~12개월Phase 2 착수: 변리사 제휴 출원 연계, 도용 대응 패키지, 첫 분쟁 대응 사례

예산 항목(공고 양식 확정 후 배분): 개발(외주·인프라) / 법률 자문·제휴 / 파일럿 운영·마케팅 / 데이터·모니터링 도구

6리스크 및 대응

수준별 평가와 대응 설계
리스크수준대응
수요 — 증명 단품은 팔리지 않음 (WIPO 선례)높음초기 공공 재원 + Phase 3 거래 연결로 지불 이유를 '보호'→'매출'로 전환
법적 효력 한계 — 등록≠권리중간"증거+출원 연계" 정직한 포지셔닝, 변리사 제휴로 정식 권리화 경로 내장
기존 대안 — 특허청 신속심사(6일)중간'공개 전 즉시 증거'와 '유통 연결'로 보완 관계 설계
규제 — 토큰화 시 금융규제 진입낮음(설계 회피)코인 미발행 원칙. 토큰화는 디지털자산기본법 2단계 입법 후 별도 검토
플랫폼 내재화 — 신상마켓 등 도매 플랫폼이 '등록 시점 증명' 기능을 자체 탑재 (딜리셔스는 2026 IPO 자금으로 AI·SaaS 확장 명시)중간방어선 = 공개 '전' 단계 + 플랫폼 중립성(신상마켓 밖 채널·오프라인 쇼룸·수출까지 커버). B2B2C 제휴로 경쟁 대신 선점

7검증 계획

제출 전 보강 목표
  • 디자이너 수요 인터뷰 3~5건 — 도용 경험, 증거 확보 습관, 등록 의향, 지불 의향 (질문지 별도 문서로)
  • 변리사 자문 1건 — 부경법 (자)목 분쟁에서 블록체인 시점 증거의 실무 효력 확인 → §2-2 근거 강화
  • 데모 영상 1~2분 — 라이브 프로토타입 화면 녹화 (등록→봉인→증명서→검증)
  • SOM 수치 구체화 — 파일럿 채널 기준 디자이너 풀 규모
